The Basics

  • Do not enter data into the gray columns of any HODL Totals sheet. This content is always calculated.
  • Enter coin inflows as positive numbers in the Net Change column.
  • Enter coin outflows as negative numbers in the Net Change column.
  • If some amount of coin entered your control, also enter that coin amount in the Acquired column.
  • If some amount of coin left your control, also enter that coin amount in the Disposed column.
  • If you know the USD value of the coin amount, set the Valuation Strategy to 'Value Known' and record this in the neighboring Value(USD) column.
  • When transfering between addresses you control, set Category to 'Transfer', Valuation Strategy to 'n/a'.

Determine Fair Market Value

  • Grab High/Lows from historical values tab on https://coinmarketcap.com.
  • If USD amount paid to receive the coin is known, enter in the Fiat Value column and set strategy to 'Value Known'.
  • If USD amount received for the coin is known, enter the in the Fiat Value column and set strategy to 'Value Known'.
  • If USD purchase/sale price per coin is known, enter in the FMV Price column and set strategy to 'Price Known'.
  • High/Low cells can contain formulas that translate sales of coin to BTC, to USD.

Calculate Your Gain/Loss

  • The transactions must be sorted into chronological order before a calculation can run.
  • The status column provides useful information for each transaction.
  • If a sale includes short and long-term components, it is split.
  • Use the Tax Doc Link column to store links to any other sheets that list or summarize the gain/loss.
