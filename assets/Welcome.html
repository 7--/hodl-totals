HODL Totals is a do-it-yourself coin and NFT tracking system based on Google Sheets. Follow the steps below to gather your transaction data into a consistent format, reconcile the sheet data with your holdings and then calculate your gains and losses.

These sheets are intended to be sufficent as an audit trail to back up your US crypto tax filing. Use this data to generate an Income Report and a 1099 Data Report (for Form 8453).

    • Do not enter data into the gray columns of any HODL Totals sheet. This content is always calculated.
    • Enter coin inflows as positive numbers in the Net Change column.
    • Enter coin outflows as negative numbers in the Net Change column.
    • If some amount of coin entered your control, also enter that coin amount in the Acquired column.
    • If some amount of coin left your control, also enter that coin amount in the Disposed column.
    • If you know the USD value of the coin amount, set the Valuation Strategy to 'Value Known' and record this in the neighboring Value(USD) column.
    • When transferring between addresses you control, set Category to 'Transfer', Valuation Strategy to 'n/a'.
    • For each transaction, provide wallet/account information and transaction IDs.
    • Use the Tx ✔ columnn to track your progress while updating/reconciling each transaction.
    • Reset the HODL Totals sheet at any time to refresh the summary of your holdings.
    • Enter your holdings on the HODL Totals sheet periodically to reconcile your records with reality.
    • Grab High/Lows from historical values tab on https://coinmarketcap.com.
    • If USD amount paid to receive the coin is known, enter in the Inflow Value(USD) column and set strategy to 'Value Known'.
    • If USD amount received for the coin is known, enter the in the Outflow Value(USD) column and set strategy to 'Value Known'.
    • If USD purchase/sale price per coin is known, enter in the Fair Mkt Value Price column and set strategy to 'Price Known'.
    • Enter data for Fair Mkt Value High/Low, set the 'Avg Daily Price Variation' strategy to track value of coins with very low trading volumes.
    • Transactions must be sorted into chronological order before a calculation can run -- simply sort the Date & Time header from A→Z.
    • The status column provides useful information for each transaction.
    • Taxable or Not Taxable status gets assigned based on the categories you configure -- edit hidden Category sheets to customize.
    • When disposing short-term and long-term coin lots in a single transaction row, the transaction gets split into two rows.
    • Use the Tax Doc Link column to store links to any other sheets that list or summarize the gain/loss.
